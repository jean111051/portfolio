## Overview

This week focused on DeskDine, a web-based cafe management system. I worked on shaping the portal for different user roles, connecting the application to Supabase, and improving the workflows needed for menu management and daily operations.

## Activities

- Researched the DeskDine workflow and planned the structure of the web portal.
- Developed pages for admin, staff, and cafe staff users.
- Added role-based access flow for the different DeskDine user types.
- Connected DeskDine to Supabase for data storage and application records.
- Worked on menu management features for daily cafe operations.
- Implemented deadline handling for daily menus.
- Added automatic expiry synchronization for menus past the deadline.
- Improved the interface and usability of DeskDine management pages.
- Fixed early issues found while testing role-based workflows.
- Continued refining the project structure using Node.js, EJS, Tailwind, and Supabase.

## Reflection

DeskDine helped me understand how important role-based workflows are in real systems. Building separate experiences for admin, staff, and cafe staff made the project feel more practical because each page had to support a specific kind of user and task.
