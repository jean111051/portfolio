## Overview

This week pushed the Disaster Response and Emergency Verification project closer to a usable deployed system. Work included hosting, mobile continuation, data connection fixes, alert behavior, and tracking improvements.

![Week 7 hosted app and tracking improvements](/images/image2.png)

## Activities

- Fixed bugs in the web app.
- Refined UI design and added new functions.
- Hosted the web app in Vercel.
- Continued development of the mobile app.
- Fixed Git and Vercel problems.
- Fixed mobile app connections to Supabase and MongoDB.
- Added a Track page to monitor reports.
- Added an alert function in the mobile app.
- Fixed tracking lookup flow so report IDs resolve more reliably.
- Added fallback handling when the first tracking source fails.
- Redesigned the tracking search panel for better mobile readability.
- Improved alignment in the compact mobile tracking summary section.

## Reflection

The tracking work was a good reminder that reliability is part of design. A report ID lookup should still guide the user clearly even when the first data source does not respond as expected.
